# Proxy pattern
クラスに期待する責務全体のうち、一部の責務を肩代わりするラッパークラスを作るパターン。
- クラスのインスタンス生成に時間がかかる
- 特定のメソッド実行に時間がかかる  

などのボトルネックを抱えるクラスにたいして、  
- ボトルネック部分の実行を他の処理と分離
- 分離した処理の実行と、ボトルネックの緩和処理の実行をラッパークラスに移譲

をする。これにより、
- 本質的な処理: ラップされるクラスに集約
- 本質的でない(ボトルネックの緩和)処理: ラッパークラスに集約

…といった具合にコードをクリーンに保てる。  

ボトルネックの緩和には非同期実行(マルチスレッド)などのテクニックが利用されることが多い。
